## rez-insights-service

**Purpose**: Stores and manages AI-generated insights from ReZ Mind for user consumption and analytics.

### Overview

The insights service acts as the central repository for all AI-generated insights, recommendations, and analytics generated by the ReZ Mind processing pipeline. It receives insights from various sources including intent processing, pattern analysis, and predictive models.

### Architecture

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────────┐
│    ReZ Mind     │────▶│ insights-service │────▶│    Copilot UI       │
│  (Intent Gen)   │     │                  │     │  (User Display)      │
└─────────────────┘     └──────────────────┘     └─────────────────────┘
                              │
                              ▼
                       ┌──────────────────┐
                       │   PostgreSQL     │
                       │   insights_db    │
                       └──────────────────┘
```

### Database Schema

```sql
CREATE TABLE insights (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id),
    insight_type VARCHAR(50) NOT NULL,
    category VARCHAR(50) NOT NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    content JSONB NOT NULL,
    confidence_score DECIMAL(3,2),
    source VARCHAR(100),
    source_id VARCHAR(255),
    actionable BOOLEAN DEFAULT true,
    priority INTEGER DEFAULT 0,
    metadata JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    read_at TIMESTAMP,
    dismissed_at TIMESTAMP,
    expires_at TIMESTAMP
);

CREATE INDEX idx_insights_user_id ON insights(user_id);
CREATE INDEX idx_insights_category ON insights(category);
CREATE INDEX idx_insights_created_at ON insights(created_at DESC);
CREATE INDEX idx_insights_unread ON insights(user_id) WHERE read_at IS NULL;
```

### Insight Types

| Type | Description | Example |
|------|-------------|---------|
| `recommendation` | Actionable suggestions | "Consider adding more inventory" |
| `prediction` | Forecast-based insights | "High demand expected next week" |
| `anomaly` | Unusual pattern detection | "Unusual order spike detected" |
| `summary` | Aggregated information | "Weekly spending summary ready" |
| `alert` | Important notifications | "Low stock warning for Item X" |

### API Endpoints

#### Create Insight
```
POST /api/insights
```

**Request Body**:
```json
{
  "userId": "uuid",
  "insightType": "recommendation",
  "category": "inventory",
  "title": "Low Stock Alert",
  "description": "Several items are running low",
  "content": {
    "items": [...],
    "threshold": 10
  },
  "confidenceScore": 0.92,
  "source": "inventory-service",
  "sourceId": "alert-123",
  "priority": 1
}
```

**Response** (201 Created):
```json
{
  "id": "uuid",
  "createdAt": "2026-05-01T10:00:00Z",
  "success": true
}
```

#### Get User Insights
```
GET /api/insights/user/:userId
```

**Query Parameters**:
- `category` (optional): Filter by category
- `type` (optional): Filter by insight type
- `unreadOnly` (optional): Only return unread insights
- `limit` (optional, default: 50): Number of results
- `offset` (optional, default: 0): Pagination offset

**Response** (200 OK):
```json
{
  "insights": [...],
  "total": 125,
  "unreadCount": 15
}
```

#### Get Single Insight
```
GET /api/insights/:insightId
```

**Response** (200 OK):
```json
{
  "id": "uuid",
  "userId": "uuid",
  "insightType": "recommendation",
  "category": "inventory",
  "title": "Low Stock Alert",
  "description": "Several items are running low",
  "content": {...},
  "confidenceScore": 0.92,
  "readAt": null,
  "createdAt": "2026-05-01T10:00:00Z"
}
```

#### Mark Insight as Read
```
PATCH /api/insights/:insightId/read
```

#### Dismiss Insight
```
PATCH /api/insights/:insightId/dismiss
```

#### Bulk Operations
```
POST /api/insights/bulk/mark-read
```
```json
{
  "insightIds": ["uuid1", "uuid2"]
}
```

## rez-automation-service

**Purpose**: Rule engine for automated triggers and workflow automation based on business events.

### Overview

The automation service provides a flexible rule-based engine that listens to business events and executes automated actions based on configurable rules. It supports complex conditions, multiple action types, and comprehensive logging.

### Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Event Bus                                    │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │
│  │  order  │  │ payment │  │ wallet  │  │  hotel  │  │  other  │    │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘    │
└───────┼────────────┼────────────┼────────────┼────────────┼─────────┘
        │            │            │            │            │
        ▼            ▼            ▼            ▼            ▼
┌─────────────────────────────────────────────────────────────────────┐
│                    automation-service                                │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                   │
│  │   Rules     │  │  Executor   │  │   Logger    │                   │
│  │   Engine    │  │             │  │             │                   │
│  └─────────────┘  └─────────────┘  └─────────────┘                   │
└─────────────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ Notification│  │   Webhook   │  │    Email    │  │   API Call  │
│   Service   │  │   Handler   │  │   Service   │  │   Handler   │
└─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘
```

### Rule Structure

```json
{
  "id": "uuid",
  "name": "Customer Churn Prevention",
  "description": "Identify and engage at-risk customers",
  "enabled": true,
  "trigger": {
    "event": "order.cancelled",
    "conditions": [
      {
        "field": "order.total",
        "operator": ">",
        "value": 100
      },
      {
        "field": "user.orderCount",
        "operator": "<",
        "value": 3
      }
    ]
  },
  "actions": [
    {
      "type": "notification",
      "target": "user",
      "template": "churn_prevention_engagement",
      "priority": "high"
    },
    {
      "type": "webhook",
      "url": "https://crm.example.com/api/flag",
      "payload": {
        "userId": "{{user.id}}",
        "reason": "churn_risk"
      }
    }
  ],
  "cooldown": 86400,
  "priority": 10,
  "tags": ["retention", "automation"]
}
```

### Built-in Rules

#### 1. Customer Churn Prevention
- **Trigger**: `order.cancelled` with high-value order
- **Conditions**: User has less than 3 orders
- **Actions**:
  - Send personalized re-engagement notification
  - Flag user in CRM
  - Generate insight for sales team

#### 2. Inventory Alerts
- **Trigger**: `inventory.updated`
- **Conditions**: Stock below threshold
- **Actions**:
  - Notify inventory manager
  - Trigger reorder workflow
  - Update dashboard metrics

#### 3. Dynamic Pricing
- **Trigger**: `market.demand_changed`
- **Conditions**: Demand exceeds 150% of normal
- **Actions**:
  - Calculate new price
  - Update product pricing
  - Log price change

#### 4. Payment Failure Recovery
- **Trigger**: `payment.failed`
- **Conditions**: Retry count < 3
- **Actions**:
  - Queue retry with exponential backoff
  - Send payment reminder to user
  - Update payment analytics

#### 5. Loyalty Points Processing
- **Trigger**: `order.completed`
- **Conditions**: User is loyalty member
- **Actions**:
  - Calculate points earned
  - Update user points balance
  - Send points notification

#### 6. Fraud Detection Alert
- **Trigger**: `order.created`
- **Conditions**: Multiple risk factors detected
- **Actions**:
  - Hold order for review
  - Alert fraud team
  - Log security event

#### 7. Welcome Series
- **Trigger**: `user.registered`
- **Conditions**: New user signup
- **Actions**:
  - Send welcome email (delayed 1 hour)
  - Create onboarding insight
  - Add to welcome campaign

#### 8. Subscription Renewal Reminder
- **Trigger**: `subscription.expiry_approaching`
- **Conditions**: 7 days before expiry
- **Actions**:
  - Send renewal reminder
  - Offer renewal incentive
  - Update CRM record

### Condition Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `==` | Equals | `status == "pending"` |
| `!=` | Not equals | `type != "test"` |
| `>` | Greater than | `total > 100` |
| `>=` | Greater or equal | `count >= 5` |
| `<` | Less than | `stock < 10` |
| `<=` | Less or equal | `priority <= 3` |
| `contains` | String contains | `name contains "urgent"` |
| `in` | Value in array | `status in ["a", "b"]` |
| `not_in` | Value not in array | `type not_in ["void", "test"]` |
| `between` | Value in range | `total between [50, 200]` |
| `regex` | Regex match | `email regex "^admin@"` |

| `insight` | Generate AI insight | `type`, `category`, `content` |

## Common Patterns

### Service Communication

All services communicate via the Event Bus using JSON messages:

```json
{
  "eventId": "uuid",
  "eventType": "order.created",
  "source": "order-service",
  "timestamp": "2026-05-01T10:00:00Z",
  "version": "1.0",
  "payload": {
    "orderId": "uuid",
    "userId": "uuid",
    "total": 150.00,
    "items": [...]
  },
  "metadata": {
    "correlationId": "uuid",
    "causationId": "uuid"
  }
}
```

### Error Handling

All services implement standardized error responses:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request parameters",
    "details": [
      {
        "field": "userId",
        "message": "userId is required"
      }
    ],
    "requestId": "uuid"
  }
}
```

### Rate Limiting

Services implement rate limiting per user:
- Read operations: 100 requests/minute
- Write operations: 20 requests/minute
- Bulk operations: 5 requests/minute

### Caching Strategy

- User insights: 5-minute cache
- Rule definitions: 1-minute cache
- Execution history: No cache (real-time)
